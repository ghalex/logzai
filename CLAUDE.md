## Repository Purpose

This is the **deployment and installation repository** for LogzAI, a log analytics platform. It contains Docker Compose configurations, installation scripts, and infrastructure configs. The actual backend source code lives in the separate `logzai-api` repository.

## Key Files

- `docker-compose.yml` - Production Docker Compose with pre-built images from Docker Hub (`ghalex/logzai-*`)
- `gateway-http.conf` / `gateway-https.conf` - Nginx reverse proxy configs
- `.env.example` - Environment variable template
- `scripts/install.sh` - Interactive installer (checks prereqs, generates secrets, pulls images, starts services)
- `scripts/update.sh` - Service updater (pulls latest images, restarts containers)
- `scripts/add-https.sh` - HTTPS/Let's Encrypt setup
- `scripts/restart.sh` - Service restart helper

## Services Architecture

The Docker Compose stack runs these services on a shared `logzai-network`:

| Service | Image | Port | Purpose |
|---------|-------|------|---------|
| `logzai-gateway` | `nginx:alpine` | 80, 443 | Reverse proxy (routes to frontend/API) |
| `logzai-frontend` | `ghalex/logzai-frontend` | 4000 | React frontend |
| `logzai-api` | `ghalex/logzai-api` | 8000 | FastAPI REST API |
| `logzai-ingestor` | `ghalex/logzai-ingestor` | 10000 | OTLP log/trace ingestion |
| `logzai-worker` | `ghalex/logzai-worker` | — | Celery background task worker |
| `logzai-beat` | `ghalex/logzai-beat` | — | Celery periodic task scheduler |
| `logzai-redis` | `redis:8.2.3-alpine` | 6379 | Cache, Celery broker |
| `logzai-db` | `postgres:16-alpine` | 5432 | PostgreSQL database |

**Data flow:** OTLP clients -> Ingestor (10000) -> Storage (S3 or local) + Redis buffer

## Common Commands

```bash
# Start/stop all services
docker compose up -d
docker compose down

# View logs
docker compose logs -f
docker compose logs -f logzai-api

# Restart a single service
docker compose restart logzai-api

# Check service status
docker compose ps

# Update a service to latest image
bash scripts/update.sh api        # or: frontend, ingestor, worker, beat, all

# Enable HTTPS
bash scripts/add-https.sh
```

## Environment Variables

Required variables in `.env` (auto-generated by `scripts/install.sh`):
- `JWT_SECRET_KEY` - JWT auth secret (generated via `openssl rand -base64 32`)
- `ENCRYPTION_KEY` - Fernet encryption key (generated via `openssl rand -base64 32`)
- `PG_DATABASE_URL` - PostgreSQL connection string
- `POSTGRES_DB`, `POSTGRES_USER`, `POSTGRES_PASSWORD` - PostgreSQL container credentials
- `REDIS_URL` - Redis connection string
- `CELERY_BROKER_URL`, `CELERY_RESULT_BACKEND` - Celery task queue config

Storage config: `USE_S3=False` uses local filesystem at `LOCAL_PATH`; `USE_S3=True` requires `S3_ENDPOINT`, `S3_BUCKET`, `S3_ACCESS_KEY`, `S3_SECRET_KEY`, `S3_REGION`.

## Install Script Behavior

`scripts/install.sh` is interactive and:
1. Checks for Docker, Docker Compose, curl, openssl
2. Checks port availability (80, 443, 8000, 10000, 6379, 5432)
3. Prompts for S3 vs local storage
4. Generates secrets (JWT, encryption key, postgres password), creates `.env`
5. Downloads configs from GitHub raw URLs
6. Pulls Docker images and starts all services
7. Waits for health checks on all services

PostgreSQL is always included as a containerized service in docker-compose.yml.
